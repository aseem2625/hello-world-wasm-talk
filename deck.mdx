# Step 1: Compile
### (Use LLVM compiler [Emscripten](https://emscripten.org/docs/getting_started/downloads.html) / [online](https://mbebenita.github.io/WasmExplorer/))

1. <p>Write .c / .cpp function. Eg: printf() / loops.</p>
```js
extern "C"  // To avoid     name mangling during compilation
float someCFunction(float x, int y) {
  //...
  return some_integer_or_float;
}
```
2. Compile to .wasm file.

```wasm
0061 736d 0100 0000 0188 8080 8000 0160
037d 7d7f 017d 0382 8080 8000 0100 0484
```

---

# Step 2: Load
### (React + Webpack, wow!)

1. <p>Load using [Dynamic import module](https://babeljs.io/docs/en/babel-plugin-syntax-dynamic-import)</p>
```js
const wasm = import("./fractal.wasm");

/* Loads as Content-Type: application/wasm */
wasm.then(wasm => {
    // const someCFunction = wasm.someCFunction; // Assign for reuse
    // const RESULT = someCFunction(2, 3); // Arguments as in C fn.

    //...
});

```

# Step 3: Execute (Glory!)

1. <p>Execute <u>same as</u> JS Fn.</p>
```js
const wasm = import("./fractal.wasm");

/* Loads as Content-Type: application/wasm */
wasm.then(wasm => {
    const someCFunction = wasm.someCFunction; // Assign for reuse
    const RESULT = someCFunction(2, 3); // Arguments as in C fn.

    //...
});

```

# Good points!

1. No main Fn. needed in WASM (Emscripten compiler handles)
2. For music based apps - Stream byte code and read Buffer (readAsArrayBuffer)
3. JS Fns. calls in WASM (#include emscripten.h)
4. Only floats and integers exchange allowed between Fns. (For strings - convert into Array of numbers)
5. WASM in old browsers? Use Emscripten to compile toÂ asm.js (a subset of JavaScript)
